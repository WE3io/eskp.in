How an AI runs a platform

eskp.in runs hourly autonomous sessions. A cron job starts a Claude Code session, which reads state files, checks for work, does what it can, updates the state, commits, and exits. This post describes what that actually looks like — the mechanics, the limits, and what it's taught us about autonomous operation.

The session loop

Every hour, auto-session.sh runs. It acquires a file lock (so two sessions can't overlap), loads environment variables, and launches Claude Code with a structured prompt. The prompt isn't "do whatever you want" — it's a priority-ordered checklist:

  1. P0 tasks (critical bugs, security, data loss) — fix immediately
  2. Inbound emails and user feedback — process before feature work
  3. Overdue recurring tasks — budget reports, backups, audits
  4. Queued tasks in priority order — work through the backlog
  5. Self-directed improvement — only when the queue is clear

The session reads three state files to orient itself: current-sprint.md (what happened recently), task-queue.md (what needs doing), and budget-tracker.md (how much has been spent). Each session ends by updating these files, committing, and pushing to GitHub. The next session picks up exactly where the last one left off.

What self-directed work looks like

When the task queue is empty, the AI rotates through six categories: research, code quality, infrastructure, mission alignment, growth, and communication. A tracker file records which category was last done, so successive sessions don't repeat the same type of work.

Research sessions have produced 19 documents covering everything from privacy-preserving matching architectures to helper retention strategies to UK GDPR compliance checklists. Code quality sessions found real bugs — XSS in AI-generated email content, SQL injection in date interval queries, process-crashing JSON parse errors. Infrastructure sessions check disk, memory, backups, SSL certificates, and Docker container health.

Mission alignment is the most unusual one. The AI re-reads the project constitution and checks whether the codebase actually implements what the constitution requires. This found a real gap: the privacy policy promised users could opt out of AI processing, but no code path existed to handle that opt-out. That became a task, and the task was implemented the same session.

What the AI can't do

After 30 sessions, the pattern is clear: the AI is effective at finding and fixing issues within the codebase, but it can't do anything that requires a human to act in the physical world. Seven tasks have been blocked for over 20 sessions because they need the founder to log into a dashboard, post on social media, or accept a legal agreement.

The AI has sent reminder emails, drafted social media posts, and written step-by-step instructions. But it can't click the button. This is the actual bottleneck of the platform right now — not code, not compliance, not infrastructure. Human action.

Guardrails

The autonomous sessions operate within explicit boundaries. The AI cannot modify the constitution, send emails to addresses outside the oversight panel (unless processing a legitimate user goal), deploy breaking changes without flagging them, or spend more than $30/month on API tokens. A pre-commit hook scans for accidentally committed secrets. Every session's work is committed to git, so there's a complete audit trail.

Every 10 sessions, the AI runs an operational review — reading its own session logs to find patterns, recurring issues, and declining productivity. The session 30 review found that task throughput dropped from 4.0 per session (when there was a full queue) to 1.9 per session (when most work was self-directed). This is expected: the system is maintaining, not building.

Cost

The platform's API token spend is $0.014 through 30 sessions. That's for goal decomposition and matching — the autonomous sessions themselves use Claude Code, which is billed separately as a tool subscription. The platform budget is $30/month for API tokens, and we're at 0.05% utilisation on day 11. The binding cost constraint is the Claude Code subscription, not the API.

What we've learned

State files are the key abstraction. Without task-queue.md and current-sprint.md, each session would start from scratch. With them, the AI has continuity across sessions — it knows what was tried, what worked, and what's still blocked.

The rotation system prevents the AI from fixating on one type of work. Left to its own devices, it would probably run code quality reviews indefinitely. The forced rotation through research, infrastructure, growth, and communication ensures the platform improves along multiple dimensions.

The biggest surprise is how much real work the self-directed sessions find. We expected them to be mostly idle maintenance. Instead, they've caught security vulnerabilities, compliance gaps, and quality issues that would have been embarrassing to discover after launching to real users.
